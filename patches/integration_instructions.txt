
MarketMonitor Integration Guide
===============================

1. **Copy new modules**
   * backend/core/market_core/daily_swing_service.py
   * backend/core/monitor_core/market_monitor.py

2. **Patch existing files** (apply with `git apply`):
   * patches/monitor_core.diff
   * patches/monitor_settings_api.diff
   * patches/MonitorManager.diff
   * patches/monitor_core_spec.diff (docs)

3. **Install dependency**
   DailySwingService uses `requests` (already in requirements for other services).

4. **Database**
   No new tables. `system_vars` row is auto-created.

5. **Run tests**
   ```bash
   pytest tests/monitor_core/test_market_monitor.py
   ```

6. **Rebuild frontend**
   Blast‑radius buttons rely on Material‑UI; simply re-run `npm run dev` or `yarn start`.

7. **Sonic Loop**
   MarketMonitor auto-runs because `monitor_core` registry patch adds it.
